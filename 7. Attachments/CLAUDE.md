# 7. Attachments — 운영 가이드

## 목적

**이미지, PDF, 오디오 등 비텍스트 파일**을 보관하는 공간입니다.  
Obsidian 설정에서 첨부 파일 기본 저장 위치를 이 폴더로 지정합니다.

## 어떤 파일이 들어가는가

| 파일 종류 | 예시 |
|----------|------|
| 이미지 | `.png`, `.jpg`, `.webp`, `.svg` |
| PDF | 논문, 전자책, 스캔 문서 |
| 오디오 | 메모 녹음 `.m4a`, `.mp3` |
| 기타 | 스프레드시트, 프레젠테이션 |

## 폴더 구조 (선택)

```
7. Attachments/
├── images/      ← 스크린샷, 다이어그램
├── pdfs/        ← 논문, 참고 문서
└── audio/       ← 녹음 파일
```

규모가 작을 때는 폴더 없이 평면 구조로 운영해도 무방합니다.

## Obsidian 설정

`설정 → 파일 및 링크 → 새 첨부 파일 위치`:
```
지정한 폴더: 7. Attachments
```

## 참조 방법

노트에서 첨부 파일은 반드시 임베드로 참조합니다:

```markdown
![[이미지파일.png]]
![[문서.pdf]]
```

경로 직접 입력 없이 Obsidian의 자동 완성을 활용합니다.

## 작업 규칙

1. **직접 편집 금지** — 이 폴더의 파일은 노트에서 참조만 합니다
2. **의미 있는 파일명** — `image001.png` 대신 `딥러닝-아키텍처-도식.png`
3. **고아 파일 정리** — 분기 1회 어떤 노트에도 참조되지 않는 파일 삭제 검토
4. **대용량 파일** — 10MB 이상 파일은 Git LFS 사용 또는 외부 링크 고려
