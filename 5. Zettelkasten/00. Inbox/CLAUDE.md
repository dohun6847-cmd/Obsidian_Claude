# 00. Inbox — 운영 가이드

## 목적

**빠른 캡처 공간**입니다. 아이디어, 생각, 링크, 인용구를 판단 없이 즉시 기록합니다.  
완성도를 요구하지 않습니다. 일단 잡아두는 것이 목적입니다.

> **규칙**: Inbox는 영구 보관소가 아닙니다. 반드시 처리됩니다.

## 어떤 것이 들어오는가

- 갑자기 떠오른 아이디어
- 나중에 읽을 글/영상 링크
- 회의나 강의 중 빠른 메모
- 책/글에서 마음에 걸리는 구절
- 아직 분류하지 못한 모든 것

## 파일 이름 규칙

```
YYYY-MM-DD_핵심단어.md
```

예시: `2025-10-15_딥러닝-직관.md`, `2025-10-15_미팅메모.md`

## 노트 형식 (최소한으로)

```yaml
---
type: inbox
created: YYYY-MM-DD
---
```

내용은 자유 형식. 완성도 신경 쓰지 않습니다.

## 정리 규칙 (주 1회)

매주 Inbox를 비웁니다. 각 노트를 보고 결정합니다:

```
각 Inbox 노트 → 
  ├── 발전시킬 가치 있음 → 10. Literature/ 또는 20. Permanent/로 이동/변환
  ├── 프로젝트/영역 관련 → 해당 PARA 폴더로 이동
  ├── 참고 자료 → 3. Resources/로 이동
  └── 불필요 → 삭제
```
